# Money as a Frozen Dataclass

> **Phase:** OOP Advanced & Modeling  •  **Stage:** 14.2 of 8  •  **Type:** `edu`  •  **Status:** skeleton (to be populated)

## What you'll learn
- Create an immutable, hashable value object with @dataclass(frozen=True, slots=True)
- Validate and normalize fields in __post_init__ (set frozen attributes via object.__setattr__)
- Use decimal.Decimal for exact monetary amounts instead of float
- Understand how frozen dataclasses auto-generate __hash__ and __eq__

## Python features introduced
`dataclasses.dataclass`, `frozen=True`, `slots=True`, `field()`, `__post_init__`, `decimal.Decimal`, `raising ValueError in validation`, `__eq__/__hash__ auto-generated by frozen dataclass`

## MiniERP increment
Introduces money.py with a Money value object (amount: Decimal, currency: str) used everywhere prices and totals appear. Replaces ad-hoc float prices from earlier phases with a precise, immutable type.
